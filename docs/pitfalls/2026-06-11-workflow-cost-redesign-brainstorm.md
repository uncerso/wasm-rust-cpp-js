# Pitfalls — 2026-06-11 workflow & cost redesign brainstorm

## Process

### Для нюансных взаимозависимых design-решений — проза с inline-опциями, не батч `AskUserQuestion` (agent-lesson)

- **What happened.** В brainstorm'е workflow-редизайна дважды подряд использовал
  батч `AskUserQuestion` (по 2–3 вопроса) на нюансных взаимозависимых темах
  (CLAUDE.md-trim + README; затем capture-timing / session-state / review-skills).
  Оба раза user отклонил, прося обсуждение — у него был контекст и возражения,
  не помещавшиеся в выбор из готовых вариантов.
- **Root cause.** `AskUserQuestion` принуждает к выбору из заданных опций. Для
  взаимозависимых решений, где user хочет внести контекст, оспорить рамку вопроса
  или уточнить, это преждевременно — выбор ещё не созрел. Инструмент хорош для
  чистых независимых форков, не для co-design нюансов.
- **Prevention.** На нюансных/взаимозависимых решениях — проза с inline-опциями и
  confidence-процентами, приглашающая к реакции. `AskUserQuestion` резервировать
  для чистых, независимых, созревших форков (где остаётся только выбрать). Симптом
  неверного применения — user отклоняет вопрос «хочу уточнить» вместо ответа.
